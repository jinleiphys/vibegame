# PI Desktop — Tenure Track

一个纯前端网页游戏。玩家扮演第 6 年理论核物理 assistant professor，在 12 周 tenure review 前，用有限经费调度 AI、小王、审稿、基金和风险审计，冲刺 3 篇 PRC 级别论文。

## 当前玩法

- 12 周回合制：每周收到 2-3 个理论核物理任务，包括 chiral EFT、SRG、少体 benchmark、相移和 HO 截断。
- 行动点系统：每周默认 3 点，派学生、跑 AI、审稿、申请 NSF、内部审计都会消耗。
- 自动导航：根据当前局面推荐并执行下一步，包括处理风险、申请基金、跑 AI、派学生或推进周结算。
- 新生入组路线：开局选择“先读文献半年”“直接跑组内旧代码”或“先复现 benchmark”，影响学生速度、质量和风险。
- 任务适配：不同任务对 AI 和学生的适配不同；推导、写作、旧代码、外推和 benchmark 的最佳处理方式不一样。
- 组会指导：花 1 个行动点主动带学生，推进 active 学生任务、加速新生训练，并降低旧代码黑箱风险。
- 本周 KPI：每周有一个短目标，完成后获得高质量成品槽、现金、声誉或下周行动点。
- 论文组装：4 个高质量成品槽合成 1 篇论文，目标是 3 篇。
- 风险系统：可疑交付、疲劳爆肝和硬刚事件会增加撤稿风险，可用内部审计处理。
- 经济系统：学生 stipend、AI 成本、publication bounty、NSF grant 和审稿收入共同影响现金流。
- 三幕剧情：中期预审、Wang 组 arXiv 抢发、预算危机、Mythos 解锁和最终 tenure review。

## 运行

直接打开 `index.html`，或在项目目录启动本地静态服务：

```bash
python3 -m http.server 4173
```

然后访问 `http://127.0.0.1:4173/index.html`。

## 主要文件

- `index.html`：页面结构和游戏窗口布局。
- `styles.css`：桌面系统、窗口、行动条、风险条和响应式样式。
- `app.js`：游戏状态、任务调度、自动导航、随机事件和结局逻辑。
